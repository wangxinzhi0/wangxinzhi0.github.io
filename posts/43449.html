1. 准备工作

1.1 注册Github

  • 在Github官网进行注册，注意要验证邮箱后方能使用，如果接收不到邮件，请设置邮箱白名单。

1.2 Git安装与配置

  • 各种操作系统（OS）安装包下载：https://git-scm.com/
  • win 安装除了安装目录其余选项默认即可，桌面右键出现Git GUI Here和Git Bash Here表示安装成功，如下图所示。

  • Git基本信息设置，在桌面右键，选择Git Bash Here，依次运行如下命令设置用户名和邮箱
git config --global user.name 'your name'  //设置为github用户名
git config --global user.email 'your email'  //设置github邮箱
git config --list  //查看设置结果
  • 运行如下命令，按几次Enter结束运行，会自动的在用户目录C:\Users\WXZ\生成一个“.ssh”的目录，里面会保存有两个文件：私钥（id_rsa）、公钥（id_rsa.pub）
ssh-keygen -t rsa -C "your email"
  • 打开github主页，点击右上角头像，选择settings，将公钥内容复制到SSH，名称随意设置，如下图

1.3 Hugo安装与配置

  • win平台到框架下载地址：https://github.com/gohugoio/hugo/releases，找到hugo_0.78.0_Windows-64bit.zip或hugo_extended_0.83.1_Windows-64bit.zip（功能更全，推荐）p)下载，解压到指定路径并添加到用户变量Path中（如果下面操作有问题就重启一下）
  • win+R打开cmd输入hugo version，出现下图表示安装成功。

恭喜你完成了所有准备工作！！！

2. Hugo本地个人博客生成

2.1 生成个人博客并设置主题

  • 在指定目录（可以是Hugo目录）右键Git Bash Here输入如下命令，会新建博客根目录
hugo new site myblog(自定义博客根目录名称)
  • 切换到根目录下(myblog)，选择Hugo主题下载，以npq为例，执行如下命令。推荐使用even主题https://github.com/olOwOlo/hugo-theme-even
git clone https://github.com/ahmedsaadxyzz/npq-hugo.git
  • 将文件移动到./myblog/themes目录下，修改名字为npq，将npq\exampleSite目录下的config.toml复制到myblog根目录，打开第5行修改为theme="npq"，修改themesdir = "../myblog/themes"
  • /themes/npq/static文件夹下放置avatar.png作为个人博客的头像

2.2 本地启动个人博客

  • 在myblog根目录下，输入下面命令在本地启动个人博客，效果图如下
  • 注意：在myblog目录下的config.toml设置好，theme、buildDrafts后，直接输入hugo server即可在本地启动个人博客。
hugo server -t npq --buildDrafts

  • 复制http://localhost:1313/，到浏览器中，查看站点是否运行成功

2.3 实际写一篇文章在本地启动

  • 切换到myblog根目录，输入如下命令，会在myblog/content/post路径下生成markdown文件，最好不要修改生成时已有内容，用vs code、Typora（强烈推荐，即时渲染markdown）或其他编辑器写markdown博客。
hugo new post/blog.md //自定义md文件名
  • 切换到myblog根目录，输入如下命令，复制http://localhost:1313/，查看本地博客内容,如下图所示。每次修改blog.md都有运行下面的命令更新内容。
hugo server -t npq --buildDrafts

恭喜你完成了本地个人博客生成！

3 将个人博客部署到远端

  • 上面的一系列操作，得到的博客网站的站点只能在自己的电脑上本地访问，要想别人能访问你的博客网站，必须将个人博客部署到远端服务器，这里以Github为例，而且仍然给出gitee的命令。
  • 在GitHub上创建新仓库，命名必须是你github的昵称且必须小写再加上.github.io，例如wangxinzhi0.github.io。Gitee的话，仓库路径为gitee名称，如wangxinzhi0，注意仓库开源！
  • 在myblog根目录下输入如下命令，配置远端地址
  • 注意：在myblog目录下的config.toml设置好，theme、baseUrl、buildDrafts后，直接输入hugo即可完成部署到远端的文件生成。
hugo --theme=even --baseUrl="https://wangxinzhi0.github.io/" --buildDrafts//配置github时
hugo --theme=even --baseUrl="https://wangxinzhi0.gitee.io/" --buildDrafts//配置gitee时
  • 然后会在myblog根目录下生成public文件夹，切换到public文件夹下，一次输入如下命令将博客推送到远端仓库
git init //初始化(只需要初始化一次)
git add . //提交全部文件
git commit -m "我的Hugo博客首次提交" //提交注释
git remote add github git@github.com:wangxinzhi0/wangxinzhi0.github.io.git  // 设置远端仓库1（只需要设置一次）
git remote add gitee git@gitee.com:wangxinzhi0/blog.git//设置远端仓库2
git push -u github master //推送到远端库1
git push -u gitee master //推送到远端库2
  • 最后，在浏览器中输入wangxinzhi0.github.io，即可访问自己的博客网站。
  • 或者wangxinzhi0.gitee.io，注意需要在仓库的服务中开启Pages服务。

已将博客部署到远端，享受吧！！！

4.常见推送问题

  • ! [rejected] master -> master (fetch first)：远端库的文件和本地库没有，先pull到本地，后push
git pull --rebase origin master
git rebase --abort //出现(master|REBASE 1/1)，回到master
git push origin master
  • ! [rejected] master -> master(non-fast-forward)
git pull origin master --allow-unrelated-histories //把远端仓库和本地同步，消除差异
重新add和commit文件
git push -u origin master
  • (master|MERGING)
git reset --hard head//回退版本信息

  如果实在无法解决冲突就直接删除public和远端仓库，重新生成public和创建github仓库，并推送到github！

+++
title = "Search🔍"
