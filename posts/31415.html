hexo博客搭建

写在前面

  虽然博主关于怎么部署个人博客写的十分清除明白，但由于软件版本迭代等原因，实际操作可能出现各种问题。自己调试了好久，终于实现了这个博客主题，真的好开心，主题是的好看，感谢博主！特此记录操作过程。

1. 环境准备

  主要安装Node.js、Hexo、Git和配置Github SSH key，及创建Github Pages页面。

  具体操作请参考这篇博客

2. 初始化Hexo项目

  主要包括安装依赖包、本地启动博客、部署到Github。 具体操作请参考这篇博客，主要命令如下：

# 初始化Hexo项目
hexo init hexoblog
# 切换到项目目录
cd hexoblog
# 安装npm包
npm i
# 本地启动Hexo博客
hexo server
# 安装hexo-deployer-git插件
npm install hexo-deployer-git --save

--------------
# 修改_config.yml文件中的url和deploy仓库地址
url: http://wangxinzhi0.github.io
deploy:
  type: git
  repo: git@github.com:wangxinzhi0/wangxinzhi0.github.io.git
  branch: master
--------------

# 部署博客到github
hexo generate
hexo deploy

  hexo init hexoblog初始化博客时可能遇到的问题

  错误一：OpenSSL SSL_read: Connection was reset, errno 10054错误，输入下方命令：

git config --global http.sslVerify "false"

  错误二：Failed to connect to github.com port 443: Timed out错误，输入下方命令取消http(s)代理：

git config --global --unset http.proxy
git config --global --unset https.proxy

  注意：如果设置完还出现上面错误，多试几次即可。

3. 设置美美的主题

  完成上面两步之后，你应该已经有一个Hexo博客了。接下来我们借助matery主题美化博客，这里使用Sitoi的github项目实现博客美化。

  • 第一步：安装依赖包，在自己博客根目录依次执行下面命令
# 安装npm依赖包
npm install
# 检测hexo版本，设置为4.1.1，版本不对会出错
hexo --version
npm i hexo@4.1.1

# hexo-abbrlink生成非中文链接
# 自动为每篇文章生成一串数字作每篇文章的 URI 地址。每篇文章的
# Front-matter 中会自动增加一个配置项：abbrlink: xxxxx，该项# 的值就是当前文章的URI地址。
npm install hexo-abbrlink --save

# 基于 hexo-generator-feed 的 Hexo 插件来做 RSS
npm install hexo-generator-feed --save

# 配置页脚信息，统计文章字数
npm i --save hexo-wordcount

# 代码高亮插件，Hexo自带高亮主题丑
npm i -S hexo-prism-plugin

# 内容搜索插件
npm install hexo-generator-search --save
  • 第二步：点击这里下载博客项目源码到本地

  将下载博客项目里的config.yml文件拷贝到自己的博客目录，根据这篇博客配置所有的config.yml内容，不需要再安装插件（所有插件上述已经安装）。

  注意：emoji表情支持插件安装失败，参考博客目录10.8指导不开启该功能即可。

  • 第三步：拷贝相关文件到自己博客中

  首先把自己博客中的scaffolds、source、themes文件夹删除（如果有博客、主题等重要内容记得备份），然后从下载博客项目中拷贝这3个文件到自己博客中。

  预览效果hexo s，如果出现ERROR Plugin load failed: hexo-theme-landscape错误，删除博客根目录node_modules/hexo-theme-landscape文件夹即可。

  • 第四步：部署到远端

  其实就是config.yml中远端仓库的设置。

  这一步可能部署到远端是空白页，仔细观察发现生成的public文件就是空的。需要卸载原有的node.js，重新安装13.5.0版本的node.js。在博客目录执行命令hexo clean && hexo g && hexo d，问题就解决了。

  • 第五步：一些自己的设置

4. 文章Front-matter介绍

  Front-matter 选项中的所有内容均为非必填的。但我仍然建议至少填写 title 和 date 的值。

  示例：

title: 基于 Hexo GitHub 从零开始搭建个人博客
date: 2019-12-30 09:25:00
author: Sitoi
img: /source/images/xxx.jpg
top: true
cover: true
coverImg: /images/1.jpg
toc: false
mathjax: false
summary: 这是你自定义的文章摘要内容
categories: Hexo
keywords: Hexo GitHub blog
tags:
    - Hexo
    - Blog
    - GitHub
配置选项 默认值 描述
title Markdown 文件标题 文章标题，强烈建议填写此选项
date 文件创建时的日期时间 发布时间，强烈建议填写此选项，且最好保证全局唯一
author 跟 _config.yml 中的 author 文章作者
img featureImages 中的某个值 文章特征图，推荐使用图床 (腾讯云、七牛云、又拍云等) 来做图片的路径。如: http://xxx.com/xxx.jpg
top true 推荐文章（文章是否置顶），如果 top 值为 true，则会作为首页推荐文章
cover false 表示该文章是否需要加入到首页轮播封面中
coverImg 无 表示该文章在首页轮播封面需要显示的图片路径，如果没有，则默认使用文章的特色图片
password 无 文章阅读密码，如果要对文章设置阅读验证密码的话，就可以设置 password 的值，该值必须是用 SHA256 加密后的密码，防止被他人识破。前提是在主题的 config.yml 中激活了 verifyPassword 选项
toc true 是否开启 TOC，可以针对某篇文章单独关闭 TOC 的功能。前提是在主题的 config.yml 中激活了 toc 选项
mathjax false 是否开启数学公式支持，本文章是否开启 mathjax，且需要在主题的 _config.yml 文件中也需要开启才行
summary 无 文章摘要，自定义的文章摘要内容，如果这个属性有值，文章卡片摘要就显示这段文字，否则程序会自动截取文章的部分内容作为摘要
categories 无 文章分类，本主题的分类表示宏观上大的分类，只建议一篇文章一个分类
tags 无 文章标签，一篇文章可以多个标签
keywords 文章标题 文章关键字，SEO 时需要
reprintPolicy cc_by 文章转载规则，可以是 cc_by,cc_by_nd,cc_by_sa,cc_by_nc,cc_by_nc_nd,cc_by_nc_sa,cc0,noreprint 或 pay 中的一个

5.主题音频、视频部署方法

方案一：

  将本地音频、视频上传到三维推，在主页场景应用中有音频推广和视频推广，上传等待审核即可。

  审核通过之后，复制应用链接，到浏览器地址栏访问。对于视频，右键检查，你会找到src="https://xxx.mp4"链接，把它复制到config.yml首页视频设置即可。对于音频，右键检查，你会找到src="https://xxx.mp3"链接，复制链接到source/data/musics.json中，音频封面图可以在网易云等网站找到，或者自己随便指定音频封面。

方案二：

  白嫖在线视频剪辑kapwing空间（需要登录），导出视频，音频，右键检查，找到mp4和mp3链接，进行相关部署即可。
